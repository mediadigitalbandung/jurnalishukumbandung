## 4. Auth & Roles

### NextAuth Configuration
- **Provider**: Credentials only (email + password, bcryptjs 12 rounds)
- **Strategy**: JWT
- **Session max age**: 4 jam (security-hardened dari 24h default)
- **Refresh**: Setiap 30 menit aktivitas
- **Rate limit login**: 5 attempts / 15 min per IP+email

### Role Matrix

| Role | Akses |
|---|---|
| **SUPER_ADMIN** | Full akses: users, settings, ads, semua panel (25 halaman) |
| **EDITOR** | Approve artikel, kelola ads, keywords, kategori, komentar |
| **JOURNALIST** | Publish langsung, create/edit artikel |
| **CONTRIBUTOR** | Create/edit artikel (draft only, submit for review) |

Helper functions di `src/lib/auth.ts`:
- `canPublishDirectly(role)` → SUPER_ADMIN, EDITOR, JOURNALIST
- `canApproveArticles(role)` → SUPER_ADMIN, EDITOR
- `canManageUsers(role)` → SUPER_ADMIN only
- `canManageAds(role)` → SUPER_ADMIN, EDITOR

### Middleware
`src/middleware.ts` melindungi `/panel/*` — redirect `/login?callbackUrl=...` jika token invalid atau user `isActive=false`.

### Pattern Standar
```typescript
export async function POST(req: NextRequest) {
  try {
    const session = await requireAuth();
    await requireRole(["SUPER_ADMIN"]);
    const data = schema.parse(await req.json());
    const result = await prisma.model.create({ data });
    await logAudit(session.user.id, "CREATE", "entity", result.id);
    return successResponse(result, 201);
  } catch (error) {
    return errorResponse(error);
  }
}
```

## 9. SEO System

### Indexing Pipeline
```
Article PUBLISHED
    │
    ├─→ Google Indexing API (instant, ~minutes)     [quota 200/day]
    ├─→ IndexNow (Bing, Yandex, Seznam, Naver)      [unlimited]
    ├─→ Cloudflare Cache purge                       [edge refresh]
    ├─→ Auto FAQ generation                          [schema.org FAQPage]
    ├─→ Auto Sorotan 10 angles                       [each submitted to IndexNow]
    ├─→ Twitter share                                [OAuth 1.0a]
    └─→ IG + FB publish                              [orchestrator]
```

### IndexNow Key
Hardcoded: `46c220e15eca4f9db0a70049aa82a734` (verified at `/46c220e15eca4f9db0a70049aa82a734.txt`)

### Toggle Flags (systemSetting)
- `google_indexing_enabled`: on/off auto-submit Google (quota 200/day)
- `google_credentials_json`: Service Account JSON (Indexing + Search Console scopes)

### SEO Checklist per Article
- ✅ `seoTitle` 50-60 char
- ✅ `seoDescription` 150-155 char
- ✅ `faqData` (auto-gen 4-5 FAQ) → FAQPage schema
- ✅ `readTime` calculated (WPM 200)
- ✅ Internal linking via `injectContextualLinks` (max 8, first-occurrence)
- ✅ OG image dynamic via `/berita/[slug]/opengraph-image`
- ✅ news_keywords meta tag + article:published_time (Jakarta ISO)

## 10. Social Media System

### Platforms Supported
- **Instagram** — Meta Graph API v21.0 (container + media_publish)
- **Facebook** — Meta Graph API v21.0 (feed link_share atau photos)
- Twitter (via OAuth 1.0a)

### Publishing Flow
```
publishArticleToSocial(articleId):
  1. Load article + settings
  2. Check autoPublishEnabled + per-article override
  3. Create SocialPost (draft atau pending)
  4. PARALLEL:
     - InstagramPublisher.prepareDraft() → postPrepared()
     - FacebookPublisher.prepareDraft() → postPrepared()
  5. Update SocialPost: success/failed + externalPostId + caption + image URL
```

### Caption Structure
```
[Title]

[Paragraf 1: Hook — intisari kasus, 2-3 kalimat]

[Paragraf 2: Konteks/implikasi, 2-3 kalimat]

📖 Lihat selengkapnya: [URL + UTM]

#Hashtag1 #Hashtag2 ... (IG: 15-20, FB: 5-8)
```

### Hashtag Priority
1. Brand (`fixedHashtagsBrand`)
2. Article tags
3. Matched `targetKeyword` (scored: exact=100, word-ratio≥70%=56-80)
4. Category
5. Platform fixed (`fixedHashtagsIg`/`fixedHashtagsFb`)

### Template System
- **Aspect ratios**: 4:5 (IG 1080×1350), 1:1 (IG 1080×1080), 1.91:1 (FB 1200×628)
- **Layers**: white base → photo (slot, cover-resized) → overlay PNG → text SVG
- **Placeholders**: `{{title}}`, `{{paraphrased_title}}`, `{{short_summary}}`, `{{category}}`, `{{date}}`, `{{author}}`
- **Output**: JPEG quality 88, mozjpeg
- **Storage**: `/public/uploads/social/`

### Draft Mode
Toggle `draftModeEnabled` di `socialMediaSettings`:
- **ON**: `prepareDraft` → DB entry (`status: draft`) → manual approve/reject via panel
- **OFF**: Langsung publish

### Delete Behavior
- **Facebook**: Support via `DELETE /{postId}?access_token=`
- **Instagram**: NO DELETE API — hanya "Tandai Dihapus" di DB (user hapus manual via IG app)
