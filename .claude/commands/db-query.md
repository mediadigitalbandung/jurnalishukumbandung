# DB-Query — Database Query Optimization Specialist

Specialist agent untuk optimasi query Prisma: N+1, missing indexes, slow queries, pagination.
Dipanggil oleh `/perf` (orchestrator) atau langsung untuk DB performance issues.

## Input

$ARGUMENTS — file atau query yang lambat. Contoh: "homepage", "artikel list", "search"

## Tugas Spesifik

Specialist ini HANYA menangani:
- Query Prisma yang lambat atau tidak efisien
- Missing database indexes
- N+1 query problems
- Unnecessary data fetching (select *)
- Pagination yang hilang atau salah
- Query di dalam loop

TIDAK menangani: bundle size (→ `/perf-bundle`), Next.js rendering strategy (→ `/perf-bundle`).

## Audit Pattern

### 1. Identifikasi Queries Bermasalah

Cari pattern ini di codebase:
```typescript
// ❌ N+1: findMany lalu per-item query
const articles = await prisma.article.findMany();
for (const a of articles) {
  const author = await prisma.user.findUnique({ where: { id: a.authorId } });
}

// ✅ Fix: include relation
const articles = await prisma.article.findMany({
  include: { author: { select: { name: true, avatar: true } } }
});
```

```typescript
// ❌ Select semua field
const articles = await prisma.article.findMany();

// ✅ Select hanya yang dibutuhkan
const articles = await prisma.article.findMany({
  select: { id: true, title: true, slug: true, publishedAt: true }
});
```

```typescript
// ❌ Tidak ada limit
const allArticles = await prisma.article.findMany({ where: { status: "PUBLISHED" } });

// ✅ Pagination
const articles = await prisma.article.findMany({
  where: { status: "PUBLISHED" },
  take: 20,
  skip: page * 20,
  orderBy: { publishedAt: "desc" }
});
```

### 2. Cek Indexes di Schema

Baca `prisma/schema.prisma`. Pastikan index ada untuk:
```prisma
// Field yang sering di-WHERE
@@index([status])
@@index([publishedAt])
@@index([categoryId])
@@index([authorId])
@@index([isAutoGenerated])

// Field yang sering di-ORDER BY
@@index([publishedAt, status])

// Field UNIQUE yang di-query
@@index([slug])  // atau @unique
```

### 3. Optimasi Promise.all untuk Parallel Queries

```typescript
// ❌ Sequential (lambat)
const articles = await prisma.article.findMany(...);
const count = await prisma.article.count(...);
const categories = await prisma.category.findMany(...);

// ✅ Parallel
const [articles, count, categories] = await Promise.all([
  prisma.article.findMany(...),
  prisma.article.count(...),
  prisma.category.findMany(...)
]);
```

### 4. Implementasi Optimasi

Terapkan fixes:
1. Tambah `@@index` ke `prisma/schema.prisma` jika missing → jalankan `npx prisma db push`
2. Refactor N+1 queries ke `include`
3. Tambah `select` untuk field spesifik
4. Tambah `take` dan `skip` untuk pagination
5. Refactor sequential queries ke `Promise.all`
