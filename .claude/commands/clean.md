# Clean — Bersihkan Data & Media JHB

Audit dan bersihkan konten bermasalah: draft lama, media orphan, artikel auto-generated non-hukum.

## Input

$ARGUMENTS — area yang ingin dibersihkan (opsional): `drafts`, `media`, `auto`, `all`

## Operasi

### drafts — Audit draft lama

Cek via panel API berapa draft yang sudah > 30 hari tidak diedit:
```
GET /api/articles?status=DRAFT&limit=100
```
Laporkan ke user: judul, tanggal dibuat, author — biarkan user yang putuskan mana yang dihapus.

### media — Cek media orphan

Cek file di direktori media yang tidak digunakan artikel manapun:
1. Baca semua `featuredImage` dan `content` dari artikel published
2. Bandingkan dengan file di `public/uploads/`
3. Laporkan file yang tidak direferensikan manapun

**JANGAN hapus otomatis** — tampilkan daftar, biarkan user konfirmasi.

### auto — Audit artikel auto-generated

Cek artikel dengan `isAutoGenerated: true` yang sudah PUBLISHED:
```
GET /api/articles?isAutoGenerated=true&status=PUBLISHED
```
Untuk setiap artikel:
- Apakah judul/konten mengandung topik hukum?
- Apakah kategori relevan dengan hukum?

Laporkan artikel yang terindikasi non-legal. Tanya user: archive atau biarkan?

### all — Jalankan semua audit di atas

Jalankan `drafts` + `media` + `auto` secara berurutan.

## Alur Default (tanpa argumen)

1. Tampilkan ringkasan statistik konten:
   - Berapa draft tertunda
   - Berapa artikel auto-generated published
   - Estimasi media orphan
2. Tanya user mau bersihkan area mana

## Aturan Penting

- **JANGAN hapus apapun tanpa konfirmasi user**
- Untuk bulk action, gunakan API `/api/articles/bulk` dengan action `archive` (bukan `delete`)
- Archive dulu, baru delete kalau user yakin
- Selalu tampilkan preview daftar sebelum tindakan apapun
